3.   2015-Atual. Identificação de Script Indiano usando Descritores de Textura.
Descrição: O principal objetivo neste trabalho é reconhecer o estilo do script indiano o qual determinado documento manuscrito foi redigido. A princípio contamos com quatro diferentes estilos de scripts, sendo eles: Bangla, Oriya, Persian e Roman. O número de documentos manuscritos difere de estilo para estilo. Nesta abordagem utilizaremos um esquema de geração de textura para a partir dos documentos originais podermos gerar uma textura mais representativa do script, para que posteriormente possamos utilizar descritores de textura como características ou mesmo uma Rede Neural Convolucional. Após gerada a textura, dividimos a imagem em blocos de textura. Assim, diferentes tamanhos de blocos podem ser avaliados. O classificador SVM será empregado, utilizando a abordagem um contra todos, com kernel RBF. Neste projeto temos algumas motivações em avaliar outros classificadores como CNN.. Situação: Em andamento; Natureza: Pesquisa. Alunos envolvidos: Graduação: (1) . Integrantes: Diego Bertolini Gonçalves - Coordenador / Marcos R. Souza - Integrante.
Membro: Diego Bertolini Gonçalves.

1.   2014-Atual. Caracterização de dependências de mudança na evolução de software por meio de métricas sociais, de processo e de código
Descrição: Sistemas de software são compostos de artefatos que dependem uns dos outros. Estudos em evolução de software introduziram o conceito de dependência de mudança (co-changes) para modelar conexões evolucionárias entre artefatos que mudam juntos. Pesquisadores tem empiricamente mostrado que essas dependências de mudanças estão relacionadas com a qualidade do software. Utilizando mineração de dados de repositórios de software, trabalhos anteriores utilizam diferentes técnicas para encontrar dependências de mudança durante a evolução do software. Entretanto, poucos trabalhos forneceram informações sobre as origens de destas dependências e quais dimensões do desenvolvimento do software estão relacionadas com as dependências de mudança. Utilizando técnicas estatísticas e modelos de classificação é possível investigar quais dimensões podem auxiliar no entendimento destas dependências, e consequentemente na evolução da arquitetura do software. Espera-se, como resultado, obter informações sobre quais dimensões impactam na ocorrência de dependências de mudança. Estas informações podem ser utilizadas por desenvolvedores em trabalhos que visem à melhoria da qualidade do software e também para a criação de ferramentas de recomendação.. Situação: Em andamento; Natureza: Pesquisa. Alunos envolvidos: Mestrado acadêmico: (1) / Mestrado profissional: (1) / Doutorado: (2) . Integrantes: Reginaldo Ré - Coordenador / Igor Scaliante Wiese - Integrante / Marco Aurélio Gerosa - Integrante / José Teodoro da Silva - Integrante / Douglas Nassif Junior - Integrante / STEINMACHER, IGOR - Integrante / Fabio Kon - Integrante / Rodrigo Takashi Kuroda - Integrante / Gustavo Ansaldi Oliva - Integrante. Financiador(es): Conselho Nacional de Desenvolvimento Científico e Tecnológico - Auxílio financeiro.

9.   2013-Atual. Investigação sobre métodos de avaliação automática de programas em disciplinas introdutórias de Computação
Descrição: Este projeto aborda a questão de avaliação no escopo de disciplinas introdutórias de programação. Tradicionalmente, os estudantes têm dificuldades em compreender os conceitos abstratos de programação e possuem visões equivocadas sobre a atividade de programação. Além disso, é necessário tornar interessantes os contéudos tratados, motivando os alunos. Um dos fatores críticos para o sucesso do ensino é promover feedback apropriado e avaliar a desempenho dos estudantes de forma constante. Em programação, isto é possível com conciliação da programação com atividades de teste de software, ou seja, a execução do programa com a intenção de encontrar erros. Esta prática permite não apenas a avaliação constante, mas, ao permitir também a criação de casos de teste, promove a reflexão sobre o funcionamento dos programas e, principalmente, dos algoritmos implementados. Entretanto, para a efetiva utilização de programas e casos de teste, é necessário o desenvolvimento e utilização de ambientes automatizados para a avaliação de trabalhos práticos, reduzindo a carga para a correção e agilizando a resposta aos aprendizes e professores. Além disto, as atividades de avaliação devem ser associadas com os conteúdos teóricos ministrados em aula. O objetivo deste projeto de pesquisa é a definição um modelo para associar elementos teóricos e de avaliação. O modelo resultante desta pesquisa será incorporado ao método LODM para desenvolvimento de objetos de aprendizagem e utilizado para a geração de objetos de aprendizagem com elementos de avaliação integrados com ferramentas automáticas de avaliação.. Situação: Em andamento; Natureza: Pesquisa. Alunos envolvidos: Graduação: (1) / Mestrado profissional: (1) . Integrantes: Marco Aurélio Graciotto Silva - Coordenador. Número de orientações: 2

10.   2013-Atual. Perfilamento de Execução para Otimização Dinâmica de Código para GPU e Linguagens de Programação Paralela
Descrição: Este projeto tem o objetivo de desenvolver e utilizar novas técnicas de perfilamento por amostragem que utilizem recursos modernos de monitoramento de execução como ferramentas para coleta de perfis de execução que possam ser utilizados em processos de otimização dinâmica. É possível utilizar estes perfis de execução para detectar regiões quentes de código passíveis à paralelização por GPU de acordo com a execução do programa. Além de detectar estas regiões é possível recompilar tais regiões, anteriormente executadas pela CPU, para executar em GPU. Assim, não é necessário recompilar o código de forma alguma, o sistema encarrega-se de utilizar os recursos de paralelismo massivamente paralelo oferecido pela GPU de forma transparente. Também é possível utilizar os perfis de execução dinamicamente coletados para melhorar a execução de programas que executam em ambientes de execução paralelos. Ao reconhecer os padrões dinâmicos do fluxo de controle do programa, é possível otimizar a sequência de execução das unidades paralelas para que o escalonamento da execução aconteça de forma eficiente.. Uma característica geral dos sistemas que aplicam as técnicas citadas acima, é que cada um permite a categorização de um tipo particular de recurso (fotos, vídeos, bookmarks, etc.), de modo que um usuário que precise armazenar e organizar diferentes tipos de recursos vê-se obrigado a interagir com diferentes sistemas, aumentando ainda mais os problemas de sinonímia, polissemia e ruído nos dados, gerados pela utilização da técnica de tagging. Este projeto propõe o desenvolvimento de um sistema denominado TagManager que possa fornecer recursos para diminuir os problemas gerados pela técnica de tagging e melhorar a qualidade da recuperação da informação. A proposta é que o sistema forneça recursos para centralizar e gerenciar o vocabulário de tags de usuários de outros sistemas baseados em tagging (uma abordagem focada nas tags) e utilize o conceito de autoridade cognitiva desses usuários como critério para definir a relevância da informação recuperada (uma abordagem focada nos usuários). Com a abordagem focada nas tags o usuário passa a ter uma visão geral de seus recursos e de suas tags em um único sistema, em vez de uma visão fracionada e limitada em cada um dos sistemas específicos. Em relação à abordagem focada no usuário, o sistema deverá suportar uma rede social, aqui chamada de rede de autoridades, que será criada por meio da concessão de autoridade cognitiva pelas pessoas e para as pessoas por meio do uso de tags.
